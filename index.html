Internet 1.0.1
- HTML: language in which elements in the internet are written.
- Servers: computers specialized in storing websites and sending these information rather than displaying it to users directly
- Browser: front end that lets user experience websites written in html code
- HTTP: The protocol used to exchange information between different computers

Information is not displayed the way it is organized in the editor. A line break in the editor does not equal a line break on the displayed website. Such matters of "design" are solved by tags like the < br > Tag

Tags:
- Come in the form of < tag > content < /tag > or just plain < tag >, when there is no content to which the tag refers (= void tag)
- Can be inline (e.g. < br > to insert a line break, < b > bold, to make text bold < /b> , < em > to make text italic < /em >) or block (e.g. < p > content < /p > or < form > content < /form >
- Are not displayed on the webpage and are only aimed at the computer interpreting it to display the actual content in the way I as a programmer want it to
- Can have attributes; e.g. the < img >-Tag for pictures has the attribute of a web adress where the respective picture is located:
- Because computers are “stupid”, they only understand correct code and are not able to correct mistakes made by the programmer, e.g. a missing > at the end of a tag
